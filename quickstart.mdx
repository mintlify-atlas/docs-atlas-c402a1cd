---
title: "Quickstart: configure PBX and make your first call"
description: "Configure your PBX server, set up user credentials, and make your first VoIP call from Odoo in under 10 minutes with voip_oca."
---

This guide walks you through the minimum configuration needed to connect Odoo to your PBX and place a call. It assumes `base_phone` and `voip_oca` are already installed. If they are not, see [Installation](/installation) first.

## Before you begin

You need the following information from your PBX provider or administrator:

- The PBX **domain** (e.g. `sip.yourprovider.com`)
- The PBX **WebSocket server URL** (e.g. `wss://sip.yourprovider.com`)
- A SIP **username** and **password** for each Odoo user who will make calls

## Configuration steps

<Steps>
  <Step title="Enable developer mode">
    The PBX server configuration menu is only visible in developer mode.

    Go to **Settings**, scroll to the bottom, and click **Activate the developer mode**. You can also append `?debug=1` to the current URL.
  </Step>

  <Step title="Create a PBX server">
    Go to **Settings > Technical > Discuss > PBX Servers** and click **New**.

    Fill in the following fields:

    | Field | Model field | Description |
    |---|---|---|
    | Name | `name` | A label for this PBX entry, e.g. `Production PBX` |
    | Domain | `domain` | The SIP domain of your PBX, e.g. `sip.yourprovider.com`. Defaults to `localhost`. |
    | WebSocket server | `ws_server` | The WebSocket URL used by the browser to connect to the PBX. Must start with `ws://` or `wss://`. Defaults to `ws://localhost`. |
    | Environment | `mode` | Set to **Test** or **Production**. |

    <Note>
      When **Environment** is set to **Test**, Odoo never contacts the PBX server. The SIP.js libraries are not loaded and no SIP registration is attempted. Use **Test** to verify the UI works before connecting a real PBX.
    </Note>

    Click **Save**.
  </Step>

  <Step title="Configure user VOIP credentials">
    Each Odoo user who will use the softphone needs their own SIP credentials assigned.

    **Option 1 — as an administrator:**

    Go to **Settings > Users & Companies > Users**, open the user's form, navigate to the **VOIP** tab, and fill in:

    | Field | Model field | Description |
    |---|---|---|
    | PBX Server | `voip_pbx_id` | Select the PBX server created in the previous step. |
    | Username | `voip_username` | The SIP username for this user, as provided by the PBX. |
    | Password | `voip_password` | The SIP password for this user. |

    **Option 2 — as the user:**

    Each user can set their own credentials by going to their **Preferences** (top-right avatar menu) and filling in the same fields under the **VOIP** tab.

    <Info>
      All three fields (`voip_pbx_id`, `voip_username`, `voip_password`) are listed in `SELF_READABLE_FIELDS` and `SELF_WRITEABLE_FIELDS`, so users can read and update their own credentials without administrator access.
    </Info>
  </Step>

  <Step title="Open the softphone widget">
    Log out and log back in to Odoo, or simply reload the page. When the page loads, Odoo reads your VOIP credentials and attempts to register with the PBX automatically.

    Look for the **softphone button** in the top navigation bar. Click it to open the softphone widget at the bottom of the screen.

    The widget has three tabs:

    - **Recent calls** — call history with date, status, and duration
    - **Call activities** — Odoo activities of type "call" that are due or overdue
    - **Contacts** — all partners with a phone number set in Odoo
  </Step>

  <Step title="Make your first call">
    You can dial in two ways:

    **Click-to-dial from a partner view:**

    Open any partner (contact, customer, or vendor) record. Phone numbers are displayed in green when the softphone is connected. Click the number to dial immediately.

    **Manual dial from the widget numpad:**

    1. In the softphone widget, open the numpad (bottom-left of the widget).
    2. Enter the phone number you want to call.
    3. Click the call button (bottom-right of the widget).

    During an active call you can:
    - **Mute** the microphone
    - **Hold** the call
    - **Transfer** the call to another number
    - **End** the call
  </Step>
</Steps>

## Receiving calls

When someone calls your SIP extension, the softphone widget opens automatically. Odoo looks up the caller's number against stored partners and displays the matching contact if found. You can answer or reject the call directly from the widget.

## Troubleshooting

<AccordionGroup>
  <Accordion title="The softphone button does not appear">
    Confirm `voip_oca` is installed (check **Apps > Installed Apps**). Also verify that the user has a PBX server assigned (`voip_pbx_id` is set). If the PBX server's **Environment** is **Test**, the button still appears but no SIP registration occurs.
  </Accordion>
  <Accordion title="The softphone connects in test mode but not in production">
    Check that `ws_server` starts with `wss://` for secure connections (required by most browsers on HTTPS pages). Verify that your PBX server allows WebSocket connections from the browser's origin and that no firewall blocks the WebSocket port.
  </Accordion>
  <Accordion title="Phone numbers are not auto-formatted">
    `base_phone` formats numbers when the partner's country is set. Ensure the partner (or their company) has a **Country** assigned so the `phonenumbers` library can determine the correct national dialing prefix.
  </Accordion>
</AccordionGroup>
