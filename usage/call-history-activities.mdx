The softphone widget provides two dedicated views for reviewing past calls and managing pending phone work: the **Recent Calls** tab and the **Call Activities** tab.

## Recent calls tab

The Recent Calls tab lists all calls associated with your user account, sorted from newest to oldest.

Each entry shows:

- **Partner name** — displayed in bold if the call is linked to an Odoo partner
- **Display name** — combines the call state and partner name or phone number (e.g., "Terminated - Acme Corp")
- **Date and time** — when the call record was created
- **Duration** — computed from `start_date` and `end_date`; only shown for calls that reached the `ongoing` state

### Call states

| State | Description |
|-------|-------------|
| `calling` | Outbound call placed, not yet answered |
| `ongoing` | Call in progress; `start_date` has been set |
| `terminated` | Call ended normally; both `start_date` and `end_date` are set |
| `missed` | Incoming call not answered before the caller hung up |
| `rejected` | Incoming call explicitly declined; `end_date` is set |
| `aborted` | Call failed before connecting |

### Searching call history

Type in the search bar above the tabs while on Recent Calls. The search runs against three fields on the `voip.call` model:

```python
search_fields = [
    "phone_number",
    "partner_id.name",
    "activity_name",
]
```

Matching is case-insensitive (`ilike`). Results are further scoped to calls owned by the current user (`user_id = uid`).

### Pagination

Calls are fetched via the `get_recent_calls` server method, which accepts `offset` and `limit` parameters. The widget loads 13 records per page by default.

```python
@api.model
def get_recent_calls(self, _search, offset, limit):
    domain = [("user_id", "=", self.env.uid)]
    # ... search filter applied if _search is set
    return [
        call.format_call()
        for call in self.search(
            domain, offset=offset, limit=limit, order="create_date DESC"
        )
    ]
```

## Call activities tab

The Call Activities tab shows `mail.activity` records that meet all of the following criteria:

- Activity type has `category = "phonecall"`
- Assigned to the currently logged-in user (`user_id = uid`)
- Deadline is today or in the past (`date_deadline <= now`)

Each entry shows:

- **Main contact name** — the primary partner linked to the related document
- **Related record name** — the document the activity is attached to (e.g., lead name, order reference); only shown when it differs from the contact name
- **Summary** — the activity's optional summary text, shown in italics

<Note>
  Activities that the current user cannot read due to multi-company restrictions or access rules are automatically excluded from the list.
</Note>

### Activity actions

Clicking an activity entry opens the partner view within the widget. In addition to the standard contact actions (send email, open partner form, schedule activity), a "Related activity" section appears with four actions:

<Steps>
  <Step title="Go to document">
    Opens the Odoo record the activity is attached to — for example, a CRM lead or a sale order — in a new window.
  </Step>
  <Step title="Mark as done">
    Marks the activity as completed and removes it from the Call Activities list.
  </Step>
  <Step title="Edit activity">
    Opens the activity form for editing the deadline, summary, or other fields.
  </Step>
  <Step title="Cancel activity">
    Deletes the activity without marking it done.
  </Step>
</Steps>

### Searching activities

The search bar filters Call Activities against the following fields via the `get_call_activities` server method:

```python
search_fields = ["res_name", "summary", "date_deadline"]
```

## Scheduling a call activity

You can schedule a new phone call activity from within the widget, without leaving your current page:

<Steps>
  <Step title="Find the contact">
    Open a partner via any of the three tabs — recent calls, activities, or contacts.
  </Step>
  <Step title="Click Schedule activity">
    In the partner view, click the clock icon in the Actions section. The `mail.activity` creation form opens in a dialog.
  </Step>
  <Step title="Configure the activity">
    Set the activity type to a phonecall type, assign a deadline, and optionally add a summary. Save to create the activity on the related document.
  </Step>
</Steps>

Once created, the activity appears in the Call Activities tab when its deadline is reached.

## Call record data model

The `voip.call` model stores all history with the following key fields:

| Field | Type | Description |
|-------|------|-------------|
| `phone_number` | `Char` | The dialed or incoming number |
| `type_call` | `Selection` | `"incoming"` or `"outgoing"` |
| `state` | `Selection` | Current call state |
| `start_date` | `Datetime` | When the call was answered |
| `end_date` | `Datetime` | When the call ended |
| `partner_id` | `Many2one` | Linked Odoo partner, if matched |
| `user_id` | `Many2one` | The Odoo user who made or received the call |
| `activity_name` | `Char` | Name of any related activity, for search purposes |

Duration is not stored directly — it is derived by subtracting `start_date` from `end_date` in the frontend when rendering the call list.
