OCA Connector Telephony is a collection of Odoo Community Association (OCA) modules that bring telephony capabilities directly into Odoo. Whether you need automatic phone number formatting with international standards or a full WebRTC-based softphone widget powered by SIP.js, these modules integrate cleanly with Odoo 18.0 and are compatible with multiple VOIP providers including Zerovoz, Ringover, Axivox, and OnSIP.

## What's included

<CardGroup cols={2}>
  <Card title="Base Phone" icon="phone" href="/modules/base-phone">
    Validates and auto-formats phone numbers to E.164 international standard using the `phonenumbers` library. Extends Odoo's built-in `phone_validation` module.
  </Card>
  <Card title="Voip OCA" icon="headset" href="/modules/voip-oca">
    Embeds a full SIP.js-powered softphone widget into the Odoo UI. Make and receive calls, track call history, and manage call activities without leaving Odoo.
  </Card>
</CardGroup>

## Get up and running

<Steps>
  <Step title="Install the modules">
    Add `base_phone` and `voip_oca` to your Odoo addons path and install them from the Apps menu. See [Installation](/installation) for prerequisites.
  </Step>
  <Step title="Configure your PBX server">
    In debug mode, go to **Settings > Technical > Discuss > PBX Servers** and create a server entry with your domain and WebSocket URL. See [PBX Setup](/configuration/pbx-setup).
  </Step>
  <Step title="Set up user credentials">
    Assign each user their PBX username and password via **Settings > Users** or through user Preferences. See [User Setup](/configuration/user-setup).
  </Step>
  <Step title="Start making calls">
    Log in to Odoo — the softphone connects automatically. Click any phone number in a partner view or open the softphone widget to dial. See [Making Calls](/usage/making-calls).
  </Step>
</Steps>

<Note>
  These modules require Odoo 18.0. Your VOIP provider must support WebRTC/WebSocket connections (RFC 7118). Test your provider compatibility before deploying to production.
</Note>
